System map — who owns what

High-level dependency graph

Seven top-level systems. Arrows show "reads from / calls into". Pose detection is handled entirely by the Ultraleap plugin in the Inspector — no custom gesture recognition code.

Ultraleap PoseDetector→Plugin-provided component. Fires UnityEvents for pose on/off. Zero custom code required for gesture recognition.
LeapInputBridge→Thin wrapper. Reads only what PoseDetector doesn't expose: hand world position, velocity, and palm normal each frame.
HandController (×2)→Receives pose events from PoseDetector. Reads position/velocity from LeapInputBridge. Owns HandState and drives MagicBall.
MagicBall (×2)→Single class: exists in hand during Prepare, becomes in-flight object on Fire. No separate Projectile class.
EnemyController→Reads StageManager for spawn config. Notifies on death via event.
StageManager→Orchestrates waves, shop, progression, TimeScaleManager.
ShopManager→Reads PlayerStats. Writes stat modifiers on purchase.
PlayerStats→Plain data object. All systems read; ShopManager writes.
ColorMath→Static utility. Pure functions, no MonoBehaviour. No deps.
TimeScaleManager→Singleton. Wraps Time.timeScale. Called by HandController on state entry/exit.

Each HandAnchor has two PoseDetector components — one for each pose. Their OnPoseDetected / OnPoseLost UnityEvents are wired in the Inspector to HandController.OnPalmUpDetected(), HandController.OnFistDetected(), etc. No per-frame gesture polling in code.
Key design decisions

Pose detection fully delegated to plugin

PalmUp and Fist poses are Scriptable Objects configured in the Ultraleap plugin. HandController only receives callbacks — it contains no gesture math. This removes an entire layer of fragile hand-recognition code.

MagicBall is both the held object and the projectile

There is one MagicBall per hand. It exists in the player's hand during Prepare, then detaches and flies on Fire. No separate Projectile class. MagicBall owns its full lifecycle: held → colored → in-flight → impact.

LeapInputBridge is slimmed down

With pose detection handled by the plugin, LeapInputBridge only needs to expose per-frame data the plugin doesn't surface: normalized hand position (for color), world velocity (for fire direction), and tracking status.

Enemy color drives damage via continuous HSV

Random hue at spawn. Damage is computed from hue distance to the inverse color — analog and smooth, not a zone lookup. Players learn to aim for the opposite corner of color space.

Ultraleap PoseDetector — plugin component (no custom code)

What the plugin handles for you

Add a PoseDetector component to each hand anchor. Assign a pose ScriptableObject (e.g. "PalmUp", "Fist") in the Inspector. Wire the UnityEvents in the Inspector:

// Inspector wiring — no code needed: PoseDetector (PalmUp pose) OnPoseDetected → HandController.OnPalmUpDetected() OnPoseLost → HandController.OnPalmUpLost() PoseDetector (Fist pose) OnPoseDetected → HandController.OnFistDetected() OnPoseLost → (not needed — fist is sampled with velocity check)
The Fist pose fires OnPoseDetected when the hand closes. HandController then checks if velocity also exceeds the throw threshold before triggering Fire. OnPoseLost for the fist is not needed since Fire is a one-frame event, not a sustained state.
LeapInputBridge.cs — MonoBehaviour, singleton (slimmed)

Responsibility

The plugin handles pose recognition. LeapInputBridge only exposes the per-frame continuous data that the plugin doesn't surface as events: normalized hand position within the tracking volume (which drives color), hand velocity in world space (which drives fire direction and throw detection), and tracking status.

Public API

public static LeapInputBridge Instance { get; private set; } // Called by HandController every frame during Prepare state: public Vector3 GetNormalizedPosition(HandSide side) // Returns hand position normalized 0-1 within tracking volume. // X→R, Y→G, Z→B for color mapping. // Called by HandController when fist pose fires: public Vector3 GetVelocity(HandSide side) // Returns world-space hand velocity in m/s (unscaled). public bool IsTracked(HandSide side) public enum HandSide { Left, Right }

Private config

[SerializeField] private Vector3 trackingVolumeMin; // world units, calibrated [SerializeField] private Vector3 trackingVolumeMax; // Reference to Ultraleap HandProvider or LeapServiceProvider [SerializeField] private LeapServiceProvider leapProvider;
LeapInputBridge does NOT replicate the full hand data struct from v1. It answers exactly three questions: where is the hand (normalized), how fast is it moving, and is it tracked. Everything else is the plugin's job.

HandController.cs — MonoBehaviour (one per hand)

Public fields & properties

[Header("Config")] [SerializeField] public HandSide side; [SerializeField] private float fireVelocityThreshold = 1.2f; // m/s [Header("References")] [SerializeField] private MagicBall magicBall; // Runtime — read-only from outside public HandState CurrentState { get; private set; } public Color CurrentColor { get; private set; }

Pose event callbacks — wired from PoseDetector in Inspector

// Called by PoseDetector (PalmUp) → OnPoseDetected: public void OnPalmUpDetected() { if (CurrentState == HandState.Idle) SetState(HandState.Prepare); } // Called by PoseDetector (PalmUp) → OnPoseLost: public void OnPalmUpLost() { if (CurrentState == HandState.Prepare) SetState(HandState.Idle); } // Called by PoseDetector (Fist) → OnPoseDetected: public void OnFistDetected() { if (CurrentState != HandState.Prepare) return; Vector3 velocity = LeapInputBridge.Instance.GetVelocity(side); if (velocity.magnitude >= fireVelocityThreshold) SetState(HandState.Fire); // If velocity too low: fist closed but not thrown — stay in Prepare }
No per-frame gesture polling. HandController's Update() only runs during Prepare state to sample position and update color — all state transitions are event-driven via the plugin.

Update() — runs only during Prepare

private void Update() { if (CurrentState != HandState.Prepare) return; if (!LeapInputBridge.Instance.IsTracked(side)) { SetState(HandState.Idle); return; } Vector3 norm = LeapInputBridge.Instance.GetNormalizedPosition(side); CurrentColor = new Color(norm.x, norm.y, norm.z); magicBall.SetColor(CurrentColor); // updates renderer this frame }

SetState() — internal transitions

private void SetState(HandState next) { CurrentState = next; switch (next) { case HandState.Idle: magicBall.SetHeld(false); TimeScaleManager.Instance.ReleaseBulletTime(side); break; case HandState.Prepare: magicBall.SetHeld(true); TimeScaleManager.Instance.RequestBulletTime(side); break; case HandState.Fire: Vector3 dir = LeapInputBridge.Instance.GetVelocity(side).normalized; magicBall.Launch(CurrentColor, dir, PlayerStats.Instance); // After launch, ball is in-flight — hand returns to Idle SetState(HandState.Idle); break; } }
MagicBall.cs — MonoBehaviour (single class, full lifecycle)

Design: one object, two phases

MagicBall exists on the hand anchor as a child object. During Prepare it is held (follows the hand anchor's transform). On Launch it is detached from the hand anchor, a Rigidbody is enabled, and it flies freely until it hits an enemy or expires. The same GameObject handles both phases — no instantiation on fire.

Public API

// Called by HandController during Prepare: public void SetHeld(bool held) // held=true: enable renderer, disable Rigidbody, follow hand anchor // held=false: disable renderer, keep Rigidbody disabled (Idle state) public void SetColor(Color color) // Updates material color via MaterialPropertyBlock (no allocation). // Also updates particle system color to match. // Called by HandController on Fire: public void Launch(Color color, Vector3 direction, PlayerStats stats) // 1. Detach from hand anchor (transform.SetParent(null)) // 2. Enable Rigidbody, set velocity = direction * stats.ProjectileSpeed // 3. Store color and baseDamage for impact calculation // 4. Start lifetime coroutine — despawn after stats.ProjectileLifetime seconds

Serialized fields

[SerializeField] private Rigidbody rb; [SerializeField] private Renderer ballRenderer; [SerializeField] private ParticleSystem heldParticles; // glow while in hand [SerializeField] private ParticleSystem trailParticles; // trail while in flight [SerializeField] private float returnDelay = 3f; // seconds before respawn

Impact & lifecycle

private void OnTriggerEnter(Collider other) { var enemy = other.GetComponent<EnemyController>(); if (enemy == null) return; float dmg = ColorMath.ComputeDamage(storedBaseDamage, storedColor, enemy.EnemyColor); enemy.TakeDamage(dmg); StartCoroutine(RespawnAfterDelay()); } private IEnumerator RespawnAfterDelay() { // Hide ball, play impact particles ballRenderer.enabled = false; yield return new WaitForSeconds(returnDelay); // Re-attach to hand anchor, reset state transform.SetParent(handAnchor); transform.localPosition = Vector3.zero; rb.velocity = Vector3.zero; rb.isKinematic = true; ballRenderer.enabled = false; // stays hidden until next Prepare } // Private fields set on Launch: private Color storedColor; private float storedBaseDamage; private Transform handAnchor; // set in Awake, used for re-parenting
After impact the ball returns to the hand anchor after a short delay, ready for the next Prepare cycle. This avoids instantiate/destroy overhead and gives a natural "cooldown feel" — the player can see their ball flying back.
ColorMath.cs — static utility class (unchanged)

Core functions

public static class ColorMath { // Perceptual inverse: hue shifts 180 degrees, S and V preserved. public static Color Inverse(Color c) { Color.RGBToHSV(c, out float h, out float s, out float v); return Color.HSVToRGB((h + 0.5f) % 1f, s, v); } // 0-1 hue distance: 0 = same hue, 1 = opposite (180 degrees apart). public static float HueDistance(Color a, Color b) { Color.RGBToHSV(a, out float ha, out _, out _); Color.RGBToHSV(b, out float hb, out _, out _); float d = Mathf.Abs(ha - hb); return Mathf.Min(d, 1f - d) * 2f; } // 1x damage at same color, up to maxMultiplier at inverse. public static float ComputeDamage(float baseDmg, Color projectile, Color enemy, float maxMultiplier = 3f) { float dist = HueDistance(projectile, Inverse(enemy)); float mult = Mathf.Lerp(1f, maxMultiplier, 1f - dist); return baseDmg * mult; } // HUD only — snaps continuous color to nearest named spell zone. public static SpellType ClassifyColor(Color c) { ... } } public enum SpellType { Neutral, Red, Green, Blue, Yellow, Cyan, Magenta, White }

Damage curve

Enemy color is a random hue at full saturation. Its inverse is always exactly 180° away. A perfectly matched inverse projectile deals 3× base damage. A same-color projectile deals 1× base damage. Colors in between scale linearly — no snap zones, no lookup table. Only hue matters, so the rule is simple enough to feel intuitively learnable.

HandController, MagicBall (while held), and LeapInputBridge all use Time.unscaledDeltaTime. The Rigidbody on MagicBall is only active during flight — and in-flight balls move through slowed time intentionally, which looks visually satisfying.
State transitions — event-driven (v2)
// Idle → Prepare: PoseDetector fires OnPoseDetected (PalmUp pose) → HandController.OnPalmUpDetected() → SetState(Prepare) → magicBall.SetHeld(true) → TimeScaleManager.RequestBulletTime(side) // Prepare → Idle (palm dropped): PoseDetector fires OnPoseLost (PalmUp pose) → HandController.OnPalmUpLost() → SetState(Idle) → magicBall.SetHeld(false) → TimeScaleManager.ReleaseBulletTime(side) // Prepare → Fire → Idle (throw): PoseDetector fires OnPoseDetected (Fist pose) → HandController.OnFistDetected() → velocity = LeapInputBridge.GetVelocity(side) → if velocity.magnitude >= threshold: → SetState(Fire) → magicBall.Launch(color, dir, stats) → SetState(Idle) immediately after
Per-frame color update — during Prepare only
HandController.Update() (runs only when state == Prepare, unscaled time): norm = LeapInputBridge.GetNormalizedPosition(side) CurrentColor = new Color(norm.x, norm.y, norm.z) magicBall.SetColor(CurrentColor) → MaterialPropertyBlock._Color update on ballRenderer → particle system color update HUD.UpdateColorPreview(CurrentColor, ColorMath.ClassifyColor(CurrentColor))
Ball impact → reward chain
MagicBall.OnTriggerEnter(enemy collider) [during flight phase] → ColorMath.ComputeDamage(storedBaseDamage, storedColor, enemy.EnemyColor) → enemy.TakeDamage(damage) → if health <= 0: enemy.Die() → OnDeath.Invoke(this) → EnemySpawner.OnEnemyDied() → StageManager.AwardCoins(coinReward) → if all enemies dead: OnWaveComplete.Invoke() → magicBall.StartCoroutine(RespawnAfterDelay())
Compared to v1: no Projectile.cs, no instantiation on fire. The MagicBall detaches and reattaches instead. State transitions are pose-event callbacks rather than per-frame conditionals. LeapInputBridge is only polled inside Update() during Prepare, and once on fist detection for velocity — not every frame unconditionally.
